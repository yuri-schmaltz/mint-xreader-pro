# QA & Estratégia de Testes: Xreader

Nossa estratégia de qualidade foca em garantir que documentos complexos continuem renderizando corretamente e que a interface de abas não sofra regressões.

## 1. Pirâmide de Testes
- **Unitários**: Focados na `libdocument` (ex: parsing de metadados).
- **Integração**: Testes de carregamento de backends.
- **E2E (UI)**: Testes de fluxo de usuário via AT-SPI (Dogtail).

## 2. Rodando Testes de UI Localmente
```bash
# Inicie o xreader em modo debug build
python3 test/testFileMenu.py ./builddir/shell/xreader
```

## 3. Critérios de Release (Quality Gates)
Para que um PR seja aceito, ele deve passar por:
- [ ] Build limpa sem warnings novos.
- [ ] Teste de fumaça (abrir PDF, DVI e DjVu).
- [ ] Verificar se as abas funcionam ao abrir múltiplos arquivos.
- [ ] Nenhuma regressão de performance (>10% no startup).

## 4. AT-SPI e Acessibilidade
O Xreader preza pela acessibilidade. Use ferramentas como `accerciser` para inspecionar a árvore AT-SPI e garantir que novos widgets de interface sejam navegáveis via leitor de tela.
